Decisions made this session
---------------------------
  1. Approval gate enforced at DB level via a BEFORE INSERT|UPDATE
     trigger (not just RLS) — defense in depth against the existing
     5-min grace UPDATE policy which would otherwise allow techs to
     elevate their own WO.
  2. Photos in Phase 3a are NOT passed as image blocks to Claude.
     Claude sees only a "[N photos attached]" textual hint. After
     a WO is created, server links photos to the first WO. Vision
     analysis lands in Phase 3b.
  3. Photos linked to first WO when multiple WOs created in one
     turn (edge case noted for later refinement; rare in practice).
  4. multer 2.x chosen over 1.x — 1.x has known CVEs.
  5. Empty-string env vars stripped at startup (fixes the
     ANTHROPIC_API_KEY="" shadow case discovered earlier).
