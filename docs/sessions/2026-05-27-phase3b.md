Caught + fixed bugs during the phase
------------------------------------

1. Chat history replay crashed the server on the 2nd turn of any
   conversation — role='tool' messages can't be sent back to Claude
   (Anthropic accepts only user/assistant; tool_result blocks ride
   inside user messages). Fixed: remap role on the way out + wrap
   /api/chat in try/catch so a future API quirk can't take down
   the server.

2. Undo never worked — void_work_order tried to resolve short-id
   prefix via .like() on a uuid column, which PostgREST can't filter.
   Fixed: inverted the API so undo with no arguments picks the
   caller's most recent WO within 5-min grace window (which is what
   "undo" means in conversation 99% of the time).

3. Voided WOs lingered in the admin review queue. Fixed: GET filter
   now excludes status='voided'; one-time SQL UPDATE reclassified
   the leftover stale ones as 'rejected' with auto-generated note.

4. Alvys driver Status field was being treated as employment state
   when it's actually session state (online/offline). Caught earlier
   in Phase 3a; the IsActive fallback is now in the mapper.

5. Intangles odo silently in km not miles. Caught earlier by the
   PM uploading an official odo report for cross-check. Conversion
   + canary test now in place.
