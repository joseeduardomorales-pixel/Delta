What landed in Phase 4
──────────────────────
Pure infrastructure / deploy automation. No code changes.

1. render.yaml + docs/deploy.md  (committed earlier on main)
   Render Blueprint config + step-by-step playbook for future
   deploys.

2. scripts/render-deploy.mjs
   Creates both Render services via API (POST /v1/services),
   pastes 13 env vars from api/.env. Idempotent.

3. scripts/render-stitch.mjs
   PUTs FRONTEND_ORIGIN onto delta-api and VITE_API_URL onto
   delta-web after both services have URLs.

4. scripts/render-wait.mjs
   Polls /v1/services/:id/deploys until both reach status='live'.

5. scripts/render-custom-domain.mjs
   POSTs delta.coldcargo.us as a custom domain on delta-web,
   polls until verified + SSL issued, updates FRONTEND_ORIGIN to
   the custom domain.

6. scripts/render-fix-routes-and-redeploy.mjs
   Post-deploy fix script. Adds SPA rewrite rule
   (/* → /index.html) on delta-web (missed in initial create).
   Triggers a manual deploy on delta-api (env var PUT alone didn't
   restart the container).
