Decisions made this session:
  1. Workspace MCP setup — Supabase MCP added at both project scope
     (.mcp.json) and user scope. PM authenticated via OAuth in a
     separate Claude Code session. Not used in THIS session (working
     dir mismatch) but available for future sessions running in
     ~/delta.
  2. Switched from psql to node-pg for migrations. Reason: psql not
     installed on this Mac, brew not installed; pg is npm-installable
     in seconds and stays inside the repo's dep tree.
  3. Connection method = Supabase Session Pooler + IPv4 (Shared
     Pooler toggle). Direct (db.{ref}.supabase.co) is IPv6-only on
     new Supabase projects and DNS-fails on IPv4 networks.
  4. dotenv parsing — wrote a quote-aware inline loader in the
     migrator because the DB password contains '#' which dotenv
     treats as a comment delimiter unless the value is quoted.
     api/.env now quotes PGPASSWORD; .env.example documents this.
  5. delta_meta schema for migration tracking (separate from public)
     so the public-RLS-coverage sanity check in 0001 doesn't fail
     on the runner's bookkeeping table.
  6. Permission model: tech-can-self-edit window enforced by RLS
     via wo_grace_window() = 5 minutes. Editable via a SQL ALTER
     if the policy needs to drift.
